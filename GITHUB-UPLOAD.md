# GitHub Pages upload instructions

Upload the **contents** of this folder to the repository root. Do not upload only `index.html`, and do not upload the ZIP file without extracting it.

The repository root should show:

```text
index.html
pricing.html
README.md
vercel.json
.nojekyll
assets/
  css/
    styles.css
  js/
    main.js
  images/
```

## Enable GitHub Pages

1. Open the GitHub repository.
2. Select **Settings**.
3. Select **Pages**.
4. Under **Build and deployment**, choose **Deploy from a branch**.
5. Select the `main` branch and the `/ (root)` folder.
6. Save and wait for GitHub to publish the site.

If the page appears unstyled, confirm that `assets/css/styles.css`, `assets/js/main.js`, and the complete `assets/images/` directory exist in the repository with exactly the same capitalization.
